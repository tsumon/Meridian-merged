# 安全策略

## 报告安全问题

如果你发现了安全漏洞，**请不要在公开的 Issue 中报告**。

请通过以下方式私下报告：

- 使用 GitHub 的 [Security Advisory](https://github.com/snnabb/Meridian/security/advisories/new) 功能
- 或发送邮件至仓库维护者（见仓库 Profile）

我们会在收到报告后尽快确认并处理。

## 当前安全边界

以下是 Meridian 当前的安全设计和已知限制，请在部署前了解：

### 认证

- 管理面板使用 JWT Bearer Token 认证
- 密码使用 bcrypt 哈希存储
- JWT 使用 HMAC-SHA256 签名，Token 有效期 72 小时
- 如果未设置 `JWT_SECRET` 环境变量，程序启动时会生成进程级随机密钥（重启后旧 Token 全部失效）

### 单用户

- 当前只支持一个管理员账户
- 不支持多用户、角色划分或权限隔离
- 任何持有有效 Token 的请求都可以执行所有管理操作

### 网络

- 管理面板默认监听 `0.0.0.0`，建议在生产环境中通过反向代理（如 Nginx/Caddy）限制访问来源
- 管理面板本身不提供 HTTPS，需要外层反代处理 TLS 终止
- 反代站点的监听端口同样绑定在所有接口上

### 数据

- SQLite 数据库文件包含管理员密码哈希和站点配置，注意文件权限
- 没有审计日志，操作不可追溯
- 没有请求频率限制

### 上游通信

- 与上游 Emby 服务器的通信基于配置的 URL scheme（HTTP 或 HTTPS）
- TLS 诊断在检测证书时使用了 `InsecureSkipVerify`（仅用于读取证书信息，不影响实际代理流量）

## 支持的版本

当前项目处于活跃开发阶段，安全修复仅针对最新版本。
